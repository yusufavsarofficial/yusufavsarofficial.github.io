Siber Güvenlik Odaklı Mikroservis Mimarisi Tasarımı

14 Kasım 2023 · 15 dakika okuma

ÖZET: Bu makale, modern dağıtık sistemlerdeki artan saldırı yüzeyine yanıt olarak, Sıfır Güven (Zero Trust) modelini esas alan, kapsamlı bir mikroservis mimarisi tasarım metodolojisini sunmaktadır.

I. GİRİŞ: MİMARİ RİSKİN DEĞİŞEN MANZARASI

Mikroservisler, çevikliği ve ölçeklenebilirliği getirse de, iki kritik risk alanı yarattı: Yatay Hareket (Lateral Movement) ve Bozuk Yetkilendirme Mantığı (BOLA). Geleneksel mimaride, başarılı bir dış ihlal sonrası saldırgan içeride "güvenli bölgeye" girerdi. Mikroservislerde ise, her servis arasındaki iletişimin şifrelenmemesi ve yetkilendirilmemesi, bir servisin ihlali durumunda saldırganın içeride hızla yayılmasına olanak tanır.

II. AĞ KATMANI GÜVENLİĞİ: SIFIR GÜVEN VE SERVICE MESH ANALİZİ

Sıfır Güven modelinin temel taşı mTLS uygulamasıdır. Her servisin sadece geçerli bir sertifikaya sahip olan diğer servislerle iletişim kurması zorunlu kılınır. Service Mesh (Istio, Linkerd) bu süreci otomatikleştirir.

2.1. mTLS Uygulamasının Teknik Derinliği ve Performans Kaygıları

mTLS şifreleme ve şifre çözme işlemleri CPU yükü ve gecikme (latency) yaratır. Endüstri analizlerine göre, Sidecar'ların yarattığı gecikme, ek yükü optimize etmek için Go veya Rust ile yazılmış proxy'ler (Envoy) kullanılarak genellikle 1-5 milisaniye (τmTLS) aralığında tutulur.

III. YETKİLENDİRME VE UYGULAMA MANTIĞI KONTROLÜ

API Gateway'de üretilen JWT'ler, servislere ulaştığında sadece varlığı değil, içeriği ve kullanım amacı doğrulanmalıdır. Servisler, token'ı alıp sadece imzayı (HMACsig) kontrol etmekle kalmamalı, aynı zamanda algoritma, süre (exp) ve hedef kitle (aud) gibi kriterleri de doğrulamalıdır.

3.1. BOLA Önleme Pseudosu

Aşağıdaki Go kodu, bir kaynağa erişim talebi geldiğinde, isteği yapan kullanıcının o kaynağın sahibi olup olmadığını nasıl kontrol ettiğini gösterir:


func CheckAuthorization(req *http.Request, resourceID int) error {
    // JWT'den gelen kullanıcı kimliğini al
    claims, ok := req.Context().Value("claims").(*MyClaims)
    if !ok {
        return ErrUnauthorized
    }
    userID := claims.Subject

    // Veritabanından kaynağın sahibini getir
    ownerID, err := db.GetResourceOwner(resourceID)
    if err != nil {
        return err
    }
    
    // Sahiplik kontrolü yap
    if userID != ownerID {
        // Kritik güvenlik olayını logla ve erişimi reddet
        log.Printf("SECURITY ALERT: User %s tried to access resource %d owned by %s", userID, resourceID, ownerID)
        return ErrForbidden
    }

    return nil
}

IV. SONUÇ VE ÖNERİLER

Siber güvenlik odaklı mikroservis mimarisi, geleneksel yöntemlerin ötesinde kriptografik kesinlik, minimum yetki ve zorunlu otomasyon gerektirir. Bu makalede sunulan mTLS, PaC ve sıkı yetkilendirme prensipleri, mimarinizi saldırganların yatay hareketine ve mantık zafiyetlerine karşı dayanıklı hale getirecektir.
